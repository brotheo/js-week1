🔹 What is JavaScript?

Background Note:

Imagine your webpage is like a robot.

  • HTML gives the robot its body – its arms, legs, head, etc. (the structure).
  • CSS dresses the robot up – giving it cool colors, shiny parts, and maybe even a hat! (the style).
  • But what makes the robot do things? What makes it walk, talk, or play a game? That's JavaScript!

JavaScript (JS) is like the "brain of the webpage." It makes things *happen*. It's what allows for:

  • Interactive buttons
  • Games
  • Animations
  • Pop-up messages
  • And so much more!

Adding JavaScript to Your HTML

To tell your HTML file to run JavaScript, you use the <script> tag. You can put this tag almost anywhere in your HTML, but for now, we'll put it just before the closing </body> tag so you can see it easily.

<body>
  <h1>My Awesome Webpage</h1>

  <script>
    // Your JavaScript code will go here!
  </script>

</body>

Let's Make Some Noise! (`alert()` and `console.log()`)

Two of the simplest ways to see JavaScript in action are alert() and console.log().

  • alert("Your message here!"): This creates a pop-up window in the browser with your message. It's great for getting immediate attention!
  • console.log("Your message here!"): This sends a message to the "console" in your browser's developer tools. It's super useful for checking what your code is doing behind the scenes, without annoying your users with pop-ups.

The `onclick` Attribute and Functions

HTML elements, like buttons, can have special attributes that listen for events. One very common event is onclick. When you click an element with an onclick attribute, the JavaScript code inside that attribute will run!

We'll also introduce a function. A function is like a mini-program or a set of instructions that you can name and then run whenever you want.

<button onclick="sayHi()">Click Me!</button>

Here, sayHi is the name of our function. The code inside the curly braces {} is what happens when sayHi() is called.

🔹 Hands-On Practice #2 – Let’s Personalize It!

So far, our JavaScript has been pretty set in stone. What if we want to get information *from* the user? That's where prompt() comes in!

  • prompt("Your question here?"): This creates a pop-up window that asks the user a question and allows them to type in an answer. The answer they type is then stored in a "variable."

What's a "Variable"?

Think of a variable as a named box where you can store information. You give the box a name (like name or age), and then you can put different things inside it.

In JavaScript, you create a variable using let (or const or var, but let is a good starting point):

let myBox = "Hello!"; // Now, myBox contains the text "Hello!"
